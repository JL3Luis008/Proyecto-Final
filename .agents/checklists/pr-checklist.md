# PR Review Checklist

- [ ] ¿El código cumple con el Spec relacionado?
- [ ] ¿Se han borrado todos los `console.log` de debug?
- [ ] ¿Los nombres de variables/funciones son descriptivos?
- [ ] ¿Hay documentación técnica JSDoc o similar en funciones críticas?
- [ ] ¿Se han considerado los riesgos de seguridad (SQLi, XSS, CSRF)?
- [ ] ¿El código es fáci de leer y mantener?
- [ ] ¿Se han seguido los patrones de diseño del proyecto?
- [ ] ¿Se ha adjuntado evidencia funcional válida?
